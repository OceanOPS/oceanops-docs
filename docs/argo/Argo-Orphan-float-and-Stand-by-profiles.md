# Orphan Float
The concept of an 'Orphan Float' refers to a float that has not been adopted by any DAC, and as a result, cannot be qualified in delayed mode.

To address this issue, it is now mandatory to associate every program with a DAC and designate a Core/BGC Delayed Mode Contact Person. This ensures that there is always a point of contact, either an individual or an agency, responsible for the delayed mode qualification."

# Stand by profile
Standard Argo requierments is to qualify T/S in delayed time 1 year after Real time. 
For BGC variable, this may be longer depending on the variable. 

Stand by profile refers to profile that should be qualify in delayed time according to Argo requierments, but are not yet qualified.

In order to identify those profiles that have not been DMQC after a certain period it is possible to select float based on their level of quality, by variable, considering the adjusted caracteristic too (for some parameters it is relevant to exclud adjusted value from the selection).
