# Roles and assiciated rights
Each program has contributors and each contributors have roles. This section aims at describing the responsibilities of the different roles for the different networks.

## Editing Program information:
Program manager and Operation manager have right to edit Program information.

## Editing program platform information. 
Any contributor of a program can edit platform information registered under a program.

## Argo:
In the Argo programs 3 roles are assigned with Argo specific responsibilities:
* Program manager: **mandatory** The program manager receive the EEZ notification for their floats. The program manager receive the QC notifications. The program manager receive the deployment notification of each floats.
* Operations Manager : not mandatory – useful if the Program Manager wants to delegate IOC notification feedback
* Core delayed mode contact person : **mandatory** The Core delayed mode contact person recieve the QC notifications for Core floats
* BGC delayed mode contact person : **mandatory** The BGC delayed mode contact person receive the QC notifications for BGC floats.
* Others – can be assigned any role; "Member" is the preferred option. If someone wants to assign a PI at the program level, please use the "Chief Scientist" role.
